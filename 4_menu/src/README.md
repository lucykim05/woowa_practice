#### 실행 결과 예시

```
점심 메뉴 추천을 시작합니다.

코치의 이름을 입력해 주세요. (, 로 구분)
토미,제임스,포코

토미(이)가 못 먹는 메뉴를 입력해 주세요.
우동,스시

제임스(이)가 못 먹는 메뉴를 입력해 주세요.
뇨끼,월남쌈

포코(이)가 못 먹는 메뉴를 입력해 주세요.
마파두부,고추잡채

메뉴 추천 결과입니다.
[ 구분 | 월요일 | 화요일 | 수요일 | 목요일 | 금요일 ]
[ 카테고리 | 한식 | 한식 | 일식 | 중식 | 아시안 ]
[ 토미 | 쌈밥 | 김치찌개 | 미소시루 | 짜장면 | 팟타이 ]
[ 제임스 | 된장찌개 | 비빔밥 | 가츠동 | 토마토 달걀볶음 | 파인애플 볶음밥 ]
[ 포코 | 된장찌개 | 불고기 | 하이라이스 | 탕수육 | 나시고렝 ]

추천을 완료했습니다.
```

## 설계

설계 과정에서 테스트 코드를 확인하였고, 랜덤한 카테고리, 메뉴 결과가 다음과 같이 나옴을 확인하였다.

1. 카테고리

pickNumberInRange 사용 시 숫자 나옴

2. 음식

shuffle 시 특정 음식을 앞에 둔채로 배열 정렬

따라서 Menu를 정리할 때

```js
menu = [
  {
    number: 카테고리 숫자,
    카테고리: 카테고리 이름,
    메뉴: 메뉴이름
  }
]
```

이와 같이 정리하기로 하였다. 추가로 파일 읽어오는 법, parsing도 공부하기 위해 menu.md에 정보를 저장 -> 변환하는 방식을 사용하기로 하였다.

프로그램의 진행 과정은 다음과 같다.

1. 시작 문구 출력
2. 코치 이름 입력
3. 각 코치 못 먹는 음식 이볅
4. 월~금 카테고리 결정
5. 카테고리 검증
6. 각 코치들이 먹게될 음식 결정
7. 결과 출력

기존에는 음식들에 숫자를 할당시키고, 카테고리에도 해당되는 숫자들을 넣어놔서 pickNumberInRange를 메뉴 선정에도 사용하려고 하였으나, 테스트 코드에 적합하지 않아 shuffle을 사용하는 방식을 사용하기로 하였다.

음식 선정의 순서는 다음과 같다.

1. 카테고리에 해당되는 메뉴 문자 배열 정리
2. shuffle -> 해당 코치가 먹을 수 있는지 확인
3. shuffle -> 해당 코치가 먹을 수 있는지 확인 & 이미 배열에 있는지 확인(다른 코치가 먹는지)
4. shuffle -> 반복
5. 만약 못 먹는다면 다시 shuffle 돌려야 하기에 여기서 while문이 필요하다.

현재 계획은 2가지로 나뉜다.

1. 코치별로 class를 생성 -> 못 먹는 음식 배정
2. 음식 입력 받아 저장
3. 결과 때는 코치별로 결과 가져옴

이 방법을 먼저 계획하였으나, 만약 한식 다음 한식이 나오게 된다면? 적합하지 않고 확인하는 방식이 어려울 것 같아 아래와 같이 변경하였다.

1. 코치별로 못 먹는 음식 객체로 저장
2. 요일별로 코치들이 먹게될 음식 배열로 저장
3. shuffle 시 새로운 배열 생성
4. 첫 shuffle 후 먹을 수 있으면 배열에 넣기
5. 그럼 그 뒤 shuffle 시에는 배열만 확인하면 됨
6. 또한 먹게될 음식도 저장해뒀기 때문에 해당 배열의 정보를 가져와서 보면 이미 먹은 음식인지 확인 가능함

이와 같이 변경하였다.
