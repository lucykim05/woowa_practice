## README.md

## 숫자 야구

힌트를 통해 컴퓨터가 뽑은 세 개의 숫자를 정확히 맞추는 프로그램

## 구현 기능 목록

### Model

Computer.js

- [x] constructor→ 숫자 뽑기 → 저장
- [x] 입력으로 숫자 배열이 주어졌을 때, 컴퓨터의 숫자와 비교
- [x] 숫자 비교 결과를 메세지 형식으로 변환하여 String으로 리턴

parser.js

- [x] 사용자 입력을 split 하여 리턴

userGuessValidator.js

- [x] 사용자 입력 검증 기능
  - [x] 배열 요소가 3가지 인지
  - [x] 배열 내부 값이 비었는지
  - [x] 배열 내부 값이 문자인지
  - [x] 배열 내부 값이 1~9 사이의 숫자인지
  - [x] 중복되는 숫자가 존재하는지

gameOverValidator.js

- [ ] 사용자 게임 종료 응답 검증 기능
  - [ ] 값이 비었는지
  - [ ] 값이 숫자인지
  - [ ] 1 또는 2 범위의 숫자인지

### View

input.js

- [x] 사용자 숫자 입력
- [ ] 사용자 게임 종료 시 응답 입력

output.js

- [x] 결과 출력
- [ ] 에러 메세지 출력

### Controller

baseballController.js

- [x] 숫자 야구 게임 시작 메세지 출력 요청
- [x] 컴퓨터 클래스 선언 및 숫자 뽑기 요청

- [x] 사용자 입력 요청

  - [x] 사용자 입력 파싱 요청
  - [x] 사용자 입력 검증 요청

- [x] 컴퓨터 객체에 사용자 배열 비교 요청
- [x] 비교 결과 출력 요청

- [x] 만약 3스트라이크가 아닐 시 사용자 입력 반복
- [ ] 3스트라이크일 시 게임 종료 메세지 출력
- [ ] 사용자 게임 종료 응답 요청
  - [ ] 사용자 게임 종료 응답 검증 요청
    - [ ] 만약 사용자의 응답이 1일 시 게임 재시작
    - [ ] 만약 사용자의 응답이 2일 시 게임 종료

### constants.js

- [x] 게임 시작 메세지 : `숫자 야구 게임을 시작합니다.`
- [x] 사용자 숫자 입력 메세지 : `숫자를 입력해주세요 :`
- [ ] 게임 종료 메세지 :
      `3개의 숫자를 모두 맞히셨습니다! 게임 종료`
      `게임을 새로 시작하려면 1, 종료하려면 2를 입력하세요.`

결과 메세지

- [x] 스트라이크
- [x] 볼
- [x] 낫싱

검증 종류 타입

- [x] GUESS = ‘g’
- [x] GAMEOVER = ‘o’

숫자

- [x] 숫자 배열 길이(=숫자 개수) ; LENGTH = 3
- [x] 숫자 최소 범위 ; MIN = 1
- [x] 숫자 최대 범위 ; MAX = 9

에러메세지

- [x] UNVALID_LENGTH : [ERROR] 숫자의 개수가 맞지 않습니다. 세자리 숫자를 입력해주세요.
- [x] SPACE : [ERROR] 숫자 사이에 띄어쓰기를 할 수 없습니다. 띄어쓰기를 제외하고 숫자만 입력해주세요.
- [x] STRING : [ERROR] 문자는 입력할 수 없습니다. 중복되지 않는 1부터 9사이의 숫자로 세자리를 입력해주세요.
- [x] OUT_OF_RANGE : [ERROR] 숫자의 범위가 맞지 않습니다. 1부터 9사이의 숫자로 세자리를 입력해주세요.
- [x] SAME_NUMBER : [ERROR] 숫자가 중복됩니다. 중복되지 않는 1부터 9사이의 숫자로 세자리를 입력해주세요.
