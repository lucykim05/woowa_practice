## 설계

프로그램의 실행 과정을 정리해보았고, 결과는 다음과 같다.

1. 사용자 입력
2. 커스텀 구분자 확인
3. 커스텀 구분자 검증
4. 구분자로 파싱
5. 파싱된 결과를 바탕으로 검증
6. 더하기

따라서 책임 분리에 따라 분리한 파일은 다음과 같다.

### Parser

- 구분자 정보 저장
- Input 파싱

### Validator

- 커스텀 구분자 검증
- 파싱된 결과 검증

### Matcher

- 입력 안에 커스텀 구분자 있는지 확인
- 있으면 return

### Controller

- 사용자 입력 받아옴
- 커스텀 구분자 받아옴
- 커스텀 구분자 검증돌림
- Parser에 정보 전달
- Parser 돌리고 결과 return
- 결과 검증
- 더하기

## 구현 체크리스트

### Parser

- [ ] 구분자 정보 받아와 저장
- [ ] 들어온 input 파싱 후 return

### Validator

- [ ] 커스텀 구분자 검증
  - [ ] 숫자인가
  - [ ] 공백인가
  - [ ] 길이가 1이 아닌가
- [ ] 파싱 결과 검증
  - [ ] 빈 곳이 있나(구분자 연속 or 숫자와 개수가 안맞음)
  - [ ] 숫자인가
  - [ ] 정수인가
  - [ ] 양수인가

### Matcher

- [x] 입력의 맨앞이 '//'으로 시작하는가
- [x] '\n'과 '//'의 사이에 하나만 있는가
- [x] 그 사이에 있는 애 return

### Controller

- [x] 사용자 입력 받아옴
- [x] Matcher 돌리고 결과 받아옴
- [ ] 결과 검증 후 결과 바탕으로 파서 생성
- [ ] 파서 돌림
- [ ] 파싱 결과 검증 돌림
- [ ] 더하기
