# 미션 - 자동차 경주

## 설계

일단 프로그램 실행 과정을 정리하면 다음과 같다.

1. 경주할 자동차 이름 입력
2. 자동차 이름 검증
3. 시도할 횟수 입력 -> 검증
4. 횟수만큼 게임로직 처리
5. 결과 출력

역할분리를 위해 일단 Controller는 입출력에 집중하기 위해

- 이름 입력
- 시도할 횟수 입력
- 게임 생성
- 결과 출력

의 기능만 하도록 하였고, 실제로 게임 로직을 실행시키는 Game의 경우에는

- 자동차 객체 생성
- 시도할 횟수 만큼 move
- 결과 도출 후 return

을 하도록 하였다. 그리고 실제로 Random한 넘버를 가지고 와서 이동하는 로직은 Car 객체 내부에 있도록 하였다. Car객체 내부에는

- 이름
- 현재 위치
- randomNumber 뽑아서 move

이와 같은 역할을 가지게 된다.

## 구현 체크리스트

### Controller

- [x] 자동차 이름 입력 받음 -> 검증
- [x] 시도할 횟수 입력 받음 -> 검증
- [x] 입력된 내용을 바탕으로 Game 생성
- [ ] 결과 받아서 출력

### Game

- [x] 차 객체 생성 및 저장
- [x] 시도할 횟수 입력 받아 Car 객체마다 move 실행
- [x] car들 위치 받아와서 위치들과 우승자 return

### Car

- [x] 이름
- [x] 현재 위치
- [x] 이동 로직
  - [x] 4 이상일 경우에만 +1

### InputView

- [x] 이름 입력
- [x] 시도 횟수 입력

### OutputView

- [ ] 자동차 위치 출력
- [ ] 우승자 출력

### Validator

- [ ] 자동차 이름 검증
  - [ ] 영어인가
