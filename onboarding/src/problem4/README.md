## 전체적인 흐름

1. 입력(Input)
2. 검증(Validator) -> 문자열 길이, 알파벳으로만 이루어졌는지 확인
3. 변환(convert) -> 입력받은 내용을 조건에 맞게 변환하여 return
4. 출력(Output)

## 기능요구사항

### 입력

입력은 요구사항에 없으나 연습을 위해 추가로 구현을 진행함

- [x] 입력 프롬프트 출력
- [x] 입력 받고 return

### 검증

제한사항에 나와있는 조건들로 구현함

- [x] 길이가 1 이상 1,000 이하
- [x] 알파벳과 공백으로만 이루어짐

### 변환

- [x] 문자열 내부의 각 요소들의 ASCII 수를 구함
- [x] 위의 숫자를 바탕으로 청개구리 단어(ASCII 번호)로 변환
- [x] 변환된 숫자를 이용하여 다시 문자열로 변경
- [x] 변환된 배열을 문자열로 합쳐 return

### 출력

- [x] 시작 부분(word, result, 구분선) 출력
- [x] 입력받은 내용의 변환 결과 출력
